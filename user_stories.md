# User Stories

## Doctor User Stories

### 1. View Appointments
**Description:**  
As a Doctor, I want to view my appointment schedule so that I can manage my daily consultations.

**Priority:** High

**Acceptance Criteria:**  
- Doctor can log in to the system  
- Doctor can view all appointments assigned to them  
- Appointment details include patient name, date, and time  

---

### 2. Update Appointment Status
**Description:**  
As a Doctor, I want to update the appointment status so that I can mark appointments as completed or cancelled.

**Priority:** Medium

**Acceptance Criteria:**  
- Doctor can change status  
- Status options include Scheduled, Completed, Cancelled  

---

### 3. View Patient Details
**Description:**  
As a Doctor, I want to see patient details before appointments.

**Priority:** Medium

**Acceptance Criteria:**  
- Doctor can view patient name  
- Doctor can view contact information  

---

### 4. Set Available Time Slots
**Description:**  
As a Doctor, I want to set my available time slots.

**Priority:** High

**Acceptance Criteria:**  
- Doctor can define availability by date and time  

---

### 5. Login to Portal
**Description:**  
As a Doctor, I want to securely log in to the doctor portal.

**Priority:** High

**Acceptance Criteria:**  
- Doctor must enter valid email and password  
- System authenticates credentials  

---

# Patient User Stories

### 1. Search Doctor
**Description:**  
As a Patient, I want to search for doctors by name or specialization.

**Priority:** High

**Acceptance Criteria:**  
- Patient can search by doctor name  
- Patient can search by specialization  

---

### 2. Book Appointment
**Description:**  
As a Patient, I want to book an appointment with a doctor.

**Priority:** High

**Acceptance Criteria:**  
- Patient selects doctor  
- Patient selects available time slot  

---

### 3. View Appointment History
**Description:**  
As a Patient, I want to see my past appointments.

**Priority:** Medium

**Acceptance Criteria:**  
- System displays previous appointments  
- Shows doctor name and date  

---

### 4. Cancel Appointment
**Description:**  
As a Patient, I want to cancel my appointment.

**Priority:** Medium

**Acceptance Criteria:**  
- Patient can cancel scheduled appointment  

---

### 5. Register Account
**Description:**  
As a Patient, I want to register an account.

**Priority:** High

**Acceptance Criteria:**  
- Patient enters name, email, phone  
- System creates patient profile  

---
